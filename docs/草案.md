# 同棲向け家計簿アプリ - 草案書

## このドキュメントについて

AI に指示を投げるように、アプリの要件や技術的な制約について書き殴っておく資料。

## アプリの概要

同棲しているカップルなどを対象とした、家計簿管理Webアプリ。使った金額を誰が払ったかを記録し、どちらが何円多く払っているかなどを表示してくれる。男性が多く払うケースもあるので、各支払いについては割り勘率6:4なども記録して、それを考慮してどっちが支払い不足かも計算する。

## 想定デバイス

スマホのWebブラウザ(Safari, Chrome)とパソコンのWebブラウザ(Chrome)。  
LINEなどとの連携も考える(APIやWebhook)。

## 機能要件

### メンバーの管理

メンバー管理ページからメンバーを追加できる、以下の情報を追加できる
- 名前

### カテゴリの追加

支払いに紐づけられるカテゴリを管理できる、デフォルトで以下のカテゴリがある
- 家賃
- 光熱費
- 通信費
- 食費
- その他

また各カテゴリには割り勘率を設定できる。メンバーの割り勘率の合計が100にならないと保存できない。デフォルトでは、$100/メンバー数$の値が入力されている

### 支出管理

以下の情報を登録できる

| 名称 | 説明 | 例 |
| - | - | - |
| 追加日時 | 自動入力 | |
| 支出内容 | 自由に入力 | 光回線、レストラン |
| カテゴリ | 一覧から選択 | |
| 割り勘率 | カテゴリを選択するとカテゴリの内容が反映されるが、編集できる | 50-50 |
| 金額 | 自由に入力 | ￥10,000 |
| 支払者 | メンバーから選択 | |

データベース上には以下の項目も追加される
- 清算計上日時: 初期値は null

追加した情報はテーブルで表示される。表の内容は編集可能。デフォルトで清算日がnullのものだけにフィルターされている。割り勘率の欄には割り勘率に基づいて計算された金額が一緒に表示される。

テーブルには一番左にチェックボックスがある、左上にはもちろんすべてにチェックがある。ボタン「清算計上する」を押すと、清算計上日時に日付が入る

### 支出自動追加

家賃など、特定のタイミングで必ず発生する支出を自動で追加する機能。支出管理で追加する項目に加え、以下の項目も設定できる
- 毎月/毎週/毎日
- (毎月の場合)日にち
- (毎週の場合)曜日

指定したタイミングになったら自動で支出として追加

### 清算管理

清算計上日時がnullでない支出について、集計した結果を表示する。割り勘率に基づいてそれぞれのメンバーが払うべき金額より多く払っているか少なく払っているかを表示する。  
上記集計には、以下のお金受け渡し機能も考慮され集計される。

上記の集計の下には、「お金受け渡し登録」と「お金受け渡し履歴」が表示される。

お金受け渡し登録は以下が登録される
- 受け渡し者
- 受け取り者
- 金額

お金受け渡し履歴には過去に登録した内容がテーブルで表示される

### 外部連携

将来的にLINEのbotあてにメッセージ送信すると、支出登録する仕組みや、清算の状況を定期的に知らせるなどを検討している
なので、APIやWebhookの仕組みを用意しておく

## 技術要件

### 使用言語

私からは指定なし

### 環境

- 自宅サーバーで動かすことを想定、パブリックにも公開しないため、セキュリティ制約は特になし  
- Dockerで動かすことを想定
- Postgres, RedisなどのDBを使用する場合は接続設定(URL, user, pass)は外部ファイル化  
開発環境でdocker-compose.ymlで自前DBを用意するのはもちろんok

本PCには、以下の開発言語が現在インストールされている
- python3
- nodejs

dockerはWSL2上にインストールされている
